HABIT

Home Automation Bridge for Intelligent Tasks
A local-first, LLM-native protocol and hardware ecosystem for controlling, understanding, and reasoning about smart devices.


What is HABIT?

HABIT is a lightweight system that connects physical devices to an offline AI brain (LLM), allowing users to control appliances, monitor environments, and receive intelligent feedback — all via natural language and JSON-based device schemas. It's not just a smart home system. It's a context-aware, privacy-first automation platform with reasoning at its core.


What Can HABIT Do?

  1. Turn devices on/off or set modes using plain English
  2. Understand device capabilities from JSON schemas
  3. Monitor real-time sensor data and report insights
  4. Forecast impact of user actions on energy usage
  5. Generate natural-language monthly reports
  6. Operate fully offline, with optional cloud backup

Example Use Cases

  1. “Turn on the fan and set it to low.”
  2. “Is the heater running?”
  3. “What's the cost of running the AC every night this month?”
  4. “Summarize my power usage this week.”
  5. “Don't allow the oven to run longer than 2 hours.”

System Components

Component Description Device Nodes ESP32-based modules attached to appliances HABIT Hub Raspberry Pi or edge device hosting LLM + control logic Protocol JSON schema for describing devices, capabilities, and states LLM Core Local model (Mistral, Phi2, etc.) using schema for decision making Security Offline auth, role-based access, device pairing Logging Local logs + optional cloud backup via API/cron


Getting Started

Set up your HABIT Hub Install dependencies (Python 3.10+, Llama.cpp/Ollama, Node-RED optional)
Flash a device node Upload firmware to ESP32 + connect relays/sensors Create a device schema Define device capabilities in JSON (schemas/fan_001.json) Run the local server python hub/main.py Ask your LLM: “Is the fan on?” “Turn off the heater after 10 minutes.”
